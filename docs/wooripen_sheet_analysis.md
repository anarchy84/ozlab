### 1.4 6대 인사이트

1. **티오더 페이드 ROAS 43%** — 광고비 5.95M 써서 매출 2.58M. **즉시 광고비 재배분** 후보.
2. **인터넷가입 페이드 ROAS 1,492%** — 어트리뷰션 인플레이션(매입DB 매출이 인터넷가입 광고에 묻어옴). 분리 필요.
3. **5월 페이드 광고비 0.29M으로 폭락** — 자체디비 513→7건. 이게 의도된 일시중지인지 확인 필요.
4. **인터넷사은품이 광고비 합보다 크다** — 4월 33.3M vs 페이드 23M. 사은품 ROI 트래킹 안 됨.
5. **외부DB 매입 30,000원/건 균일가 ROAS 367%** — 안정적이지만 30% 전환율 (517건 → 157건 개통).
6. **인센티브 일별집계 시트가 비어있음** — 대시보드_요약 인센티브 컬럼이 항상 0. 직원 시트에는 데이터 있는데 집계 미실행. **버그**.

### 2.3 손익_요약 (79행)

**역할**: 월말 손익 분석. CRM×서비스×상품 단위로 매출 + 지출 항목 + 순이익.

**4월 손익 분해**:

| 구분 | 금액 |
|---|---:|
| 자체 매출 | 26,623,000 (32%) |
| 토스 프리미엄 매출 | 56,885,000 (68%) |
| **총 매출** | **83,508,000** |
| 월세(관리비) | 2,000,000 (2%) |
| 인건비 | 23,480,970 (24%) |

| 인터넷사은품 (9회) | 33,286,000 (34%) |
| **총 지출** | **97,240,750** |
| **순손익** | **-13,732,750** |

**5월 손익 (~5/26)**:
- 자체 매출 4.62M (10%) / 토스 프리미엄 43.34M (90%) = **총 매출 47.96M**
- 지출 54.55M (월세 2M + 인건비 27.29M + 페이드 0.29M + 외부DB 8.16M + 인터넷사은품 16.81M)
- **순손익 -6.59M**

**핵심**:
- **토스 프리미엄 의존도 4월 68% → 5월 90%**. 자체 디비 의존도 급락.
- **인터넷사은품(고객 유치 비용)이 광고비 단일 항목보다 큰 비중** — 4월 34% > 외부DB 16% > 페이드 24%(4종 합산).
- 5월 인건비가 4월보다 16% 증가 (23.48M → 27.29M) — 인원 충원 또는 보너스 가능성.

#### View `incentive_monthly_summary` (인센티브_일별집계 대체)

```sql
-- 월×직원 평일/주말 P + 인센티브 자동 계산
CREATE VIEW incentive_monthly_summary AS
WITH p_by_product AS (
  SELECT
    ie.employee_id,
    -- 귀속일 = 상품별 룰
    CASE r.product_code
      WHEN '인' THEN ie.internet_install_date
      WHEN '테이블오더' THEN ie.tableorder_install_date
      WHEN '정수기' THEN ie.fridge_install_date
      WHEN 'VAN' THEN ie.tos_install_date
      WHEN '토스' THEN ie.tos_install_date
      ELSE ie.received_date
    END as attribution_date,
    ie.shift_type,
    -- 상품 P
    CASE r.product_code
      WHEN '인' THEN ie.qty_internet
      WHEN '티' THEN ie.qty_phone
      -- ... (11종)
    END * (CASE WHEN ie.shift_type='weekend' THEN r.weekend_p ELSE r.weekday_p END) as p
  FROM incentive_entries ie
  CROSS JOIN incentive_rates r
)
SELECT
  employee_id,
  date_trunc('month', attribution_date) as month,
  SUM(p) FILTER (WHERE shift_type='weekday') as weekday_p,
  SUM(p) FILTER (WHERE shift_type='weekend') as weekend_p,
  GREATEST(SUM(p) FILTER (WHERE shift_type='weekday') - 40, 0) * 30000 as weekday_incentive,
  SUM(p) FILTER (WHERE shift_type='weekend') * 50000 as weekend_incentive
FROM p_by_product
GROUP BY employee_id, date_trunc('month', attribution_date);
```

## 7. 결론 및 다음 단계

### 7.1 핵심 결론

1. **시트는 현재 진실의 원천(SoT)** 이며 ozlab DB는 부분 이관 상태 (광고비/리드는 완료, 매출/지출/인센티브는 진행 중).

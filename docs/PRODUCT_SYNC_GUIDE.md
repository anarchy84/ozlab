# 상품표 시트 sync 가이드

> 어드민 페이지: `/admin/settings/product-sync`
> 작성일: 2026-05-27

---

## 1. 흐름 한 눈에

```
공급사가 새 상품 / 가격 변경 알림
       ↓
담당자 — 구글 시트에 표준 헤더로 추가/수정
       ↓
어드민 → 상품 시트 sync → "🧪 미리보기" 클릭 → 결과 확인
       ↓
이상 없으면 → "🚀 동기화 실행" 클릭
       ↓
products 테이블 자동 업데이트 (insert/update)
```

## 2. 표준 헤더 (구글 시트 1행)

| 헤더 | 필수 | 설명 |
|---|---|---|
| 상품 이름 | ✅ | 식별용 이름. 같은 이름 있으면 자동 덮어쓰기 (update). |
| 분류 | ✅ | 인터넷 / CCTV / 키오스크 / 테이블오더 / 포스기 / 커넥트 / 태블릿 / 주방용 / 추가상품 / 기타. 새 분류면 자동 생성. |
| 공급사 |   | 네이버 / NIT / 토스 등. |
| 원가 |   | 우리가 사오는 가격 (부가세 포함). |
| 우리 수당 |   | 우리가 받는 마진. |
| 고객 가격 |   | 고객 판매 최종가. |
| 약정 기간 |   | 없음 / 12개월 / 24개월 / 36개월 / 48개월. |
| 월 정기 결제 |   | Y / N. |
| 월 결제 금액 |   | 월 정기 결제일 때만. |
| 단말기 종류 |   | 데스크형 / 스탠드형 / 벽걸이형 등. |
| 메모 |   | 비고, 특이사항. |

### 예시 행

| 상품 이름 | 분류 | 공급사 | 원가 | 우리 수당 | 고객 가격 | 약정 기간 | 월 정기 결제 | 월 결제 금액 | 단말기 종류 | 메모 |
|---|---|---|---:|---:|---:|---|---|---:|---|---|
| 포스기 + 커넥트 + 금전함 | 포스기 | 네이버 | 956,000 | 100,000 | 1,056,000 | 36개월 | N |  | 데스크형 | BEST 패키지 |
| NC-6000(PKG)_WH | 포스기 | 오케이포스 | 180,000 |  | 220,000 |  | N |  |  | 화이트 색상 |

## 3. 공급사 원본 양식도 그대로 OK

표준 헤더 외에 공급사가 보내준 양식 헤더도 자동 인식됨:

### NIT 양식 (출고정책 배포용)

| 시트 헤더 | 매핑되는 컬럼 |
|---|---|
| 품목명 | → 상품 이름 |
| 품목군 | → 분류 |
| 공급사 | → 공급사 |
| 판매가(기본) | → 원가 (= 1대 가격) |
| 제품설명 | → 메모 |

### 네이버 렌탈표

| 시트 헤더 | 매핑되는 컬럼 |
|---|---|
| 상품구성 | → 상품 이름 (패키지 구성) |
| 구성 | → 분류 (커넥트/태블릿/포스기/키오스크/주방용/추가상품) |
| 단가 / 렌탈가 | → 원가 (부가세 포함) |
| 일시불 | → 고객 가격 |
| 비고 | → 메모 |

→ 공급사 시트 그대로 sync 해도 자동으로 표준 컬럼에 들어감.

### 자동 무시되는 컬럼

다음 컬럼은 데이터에 저장하지 않음 (사용 안 함):
- `여신협회인증여부`, `인증일`, `인증만료일` (NIT 양식)
- `NO` (행 번호)

필요해지면 어드민에서 다시 결정.

## 4. 분류 (카테고리) 매핑 — 오즈랩 4종 + 옵션

| 시트 입력 | 내부 코드 | 비고 |
|---|---|---|
| **인터넷** / 인터넷가입 / SKT인터넷 / KT인터넷 / LG인터넷 / 광랜 / 500M / 1기가 | `internet` | 메인 — 지원금 모델 |
| 추가셋탑 / 셋탑 / 스탠다드 / ALL / WIFI 7 / WIFI / 애플셋탑 / OSS인센 | `internet_option` | 인터넷 부가 옵션 |

| **포스기** / 단말기 / 단말기(범용) / 단말기(특수) / POS | `pos` | 메인 — 단가 모델 |
| POS부가장비 / 멀티패드 / 매출전표 / 케이블 / 부수기자재 / 서명패드 / 커넥트 / 태블릿 / 주방용 | `pos_accessory` | POS 부속 |

## 4-1. 인터넷 정책서 (지원금 모델) 변환 가이드

해피 SKT 정책서 같은 외부 양식은 다음과 같이 표준 양식으로 변환:

| 원본 (SKT 정책서) | 통합 시트로 변환 |
|---|---|
| 행 "광랜 단품" · 컬럼 전국(전액현금)=280,000 | 상품이름="SKT 광랜 단품" · 분류="인터넷" · 공급사="SKT(해피)" · **원가=0** · **우리수당=280,000** · **고객가격=0** · 약정="36개월" · 메모="5/6 정책" |
| 행 "1기가+이코노미" · 680,000 | 상품이름="SKT 1기가 + 이코노미" · 분류="인터넷" · 우리수당=680,000 |
| 행 "추가셋탑 스탠다드이상" · 100,000 | 상품이름="추가셋탑 스탠다드" · 분류="인터넷-옵션" · 우리수당=100,000 |
| 행 "WIFI 7" · 20,000 | 상품이름="WIFI 7" · 분류="인터넷-옵션" · 우리수당=20,000 |
| 행 "유심정책 79요금제이상" · 550,000 | 상품이름="유심결합 79요금제+" · 분류="인터넷-유심결합" · 우리수당=550,000 |

**중요 — 인터넷 가격 모델**:
- **원가** = 0 (공급사가 무상 제공)
- **우리 수당** = 공급사가 주는 지원금 → 우리 매출
- **고객 가격** = 0 (고객 무료)
- 시점별 가격 변경은 동일 상품명으로 최신값 덮어쓰기 (update)

## 4-2. 단말기/CCTV/키오스크 (단가 모델)

| 항목 | 의미 |
|---|---|
| **원가** | 우리가 사오는 가격 (부가세 포함) |
| **우리 수당** | 마진 (선택) |
| **고객 가격** | 판매가 (= 원가 + 마진) |

## 5. 시트 권한 설정

구글 시트 → 우측 상단 **공유** → **링크가 있는 모든 사용자 — 뷰어** 로 설정.
(이게 안 되어 있으면 sync 시 HTML 응답 받아서 에러 메시지 표시됨.)

## 6. URL 입력

시트의 어떤 URL이든 OK:
- `https://docs.google.com/spreadsheets/d/.../edit?usp=sharing` (편집 URL)
- `https://docs.google.com/spreadsheets/d/.../edit#gid=123456` (탭 지정)
- `https://docs.google.com/spreadsheets/d/.../export?format=csv&gid=123456` (이미 변환된 URL)

→ sync API가 자동으로 CSV export 형식으로 변환.
→ 여러 탭이 있는 경우 URL 끝의 `#gid=` 번호로 탭 지정.

## 7. 미리보기 → 실제 적용 흐름

**1단계: 🧪 미리보기 (검증만)**
- DB 변경 X
- 각 행이 insert 될지 / update 될지 / 에러인지 결과만 표시
- 에러 행이 있으면 보고 시트 수정

**2단계: 🚀 동기화 실행**
- 미리보기 통과 후 실제 적용
- 에러 행이 한 건이라도 있으면 **전체 거부** (안전)
- 성공 시 상품 관리 페이지에서 즉시 확인 가능

## 8. 자주 발생하는 에러

| 메시지 | 원인 | 해결 |
|---|---|---|
| `상품 이름이 비어 있음` | 필수 컬럼 누락 | 시트의 빈 행 삭제 또는 상품 이름 입력 |
| `분류가 비어 있음` | 필수 컬럼 누락 | 분류 입력 |
| `약정 기간은 ... 중 하나` | 잘못된 값 (예: "1년") | "12개월" / "24개월" / "36개월" 등 표준값 사용 (또는 매핑 — "1년" → 12개월 자동) |
| `시트 공유 권한 미설정` | 비공개 시트 | "링크가 있는 모든 사용자 — 뷰어" 로 변경 |
| `CSV 행 0건` | 빈 시트 or 잘못된 gid | 시트에 데이터 있는지 + URL의 gid 확인 |

## 9. 운영 팁

- **시트는 1개만 권장** — 모든 공급사 상품을 한 시트에 통합. 매주/매월 정리.
- **마지막 sync 결과**는 어드민 페이지 하단에 항상 표시됨. 신규 N건 / 업데이트 N건 확인.
- **변경 이력 보존** — products 테이블의 updated_at 으로 마지막 변경 시각 추적. 별도 audit log 는 추후 필요 시 추가.
- **상품 삭제** — sync 는 insert/update 만. 삭제는 어드민 상품 관리 페이지에서 수동 (is_active=false 권장).

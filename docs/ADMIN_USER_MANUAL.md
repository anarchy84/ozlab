## 1. 시작하기

### 1.1 서비스 한 줄 요약

자영업자에게 POS·카드 단말기·네이버페이·리뷰 자동화를 묶어 제공하는 서비스의 신청 접수와 마케팅을 한 화면에서 처리하는 어드민입니다.

### 1.2 어드민 진입 주소

| 환경 | URL |
|---|---|
| 라이브 사이트 (사용자가 보는 화면) | https://ozlabpay.kr |
| 어드민 진입 | https://ozlabpay.kr/admin |
| 로그인 페이지 | https://ozlabpay.kr/admin/login |
| 블로그 (사용자 노출용) | https://ozlabpay.kr/blog |

### 1.3 로그인 방법

1. https://ozlabpay.kr/admin/login 접속
2. 본인 이메일과 비밀번호 입력 → "로그인"
3. 로그인 성공 시 대시보드(`/admin`)로 자동 이동

> **계정이 없으신가요?** 회원가입은 차단되어 있습니다. super_admin(대표)이 직접 초대해야 계정이 생성됩니다. 7번 섹션 참고.

### 1.4 권한 등급 5종 (v1.2 재설계)

| 권한 | 표시 | 할 수 있는 일 |
|---|---|---|
| **super_admin** | 👑 슈퍼어드민 | 모든 권한 + 사용자 추가/삭제 + 권한 매트릭스 편집 |
| **marketing** | 📊 마케팅팀 | 광고/매출/콘텐츠/CTA/상태/상담 (계정 관리 제외) — super_admin과 거의 동등 |
| **tm_lead** | 🎧 TM실장 | 상담 운영 총괄 — super_admin과 거의 동등 (계정 관리 제외) |
| **counselor** | 📞 상담사 | 상담 처리·매출 입력 (계정 초대·DB 분배 불가) |
| **it_ops** | 🛠 전산관리자 | 시스템 운영 (광고·매출·지출 안 보임) |

> 💡 **핵심**: 권한은 super_admin이 `/admin/settings/permissions` 에서 **언제든 줬다 뺐다 가능**. role 자체는 라벨일 뿐이고, 실제 가능한 동작은 권한 매트릭스에서 결정.

> ⚠️ **레거시 role**: 기존 admin/marketer/viewer 도 호환을 위해 유지. 점진 이전 권장 (admin → marketing 또는 tm_lead).

> 권한은 super_admin이 부여합니다. 본인 권한이 헷갈릴 땐 어드민 우상단 이름 옆 라벨로 확인할 수 있습니다.

## 2. 메인 페이지 인라인 편집

홈페이지(`https://ozlabpay.kr`)의 텍스트와 이미지를 **워드프레스처럼 페이지에서 직접 편집**할 수 있습니다.

### 2.1 활성화 조건

- super_admin / admin / marketer 권한으로 어드민에 로그인된 상태여야 합니다.
- 같은 브라우저에서 https://ozlabpay.kr 으로 이동하면 편집 모드가 자동 활성화됩니다.

### 2.2 텍스트 편집

1. 홈페이지에서 편집할 문장에 **마우스를 가져갑니다** (예: 메인 카피 "결제부터 리뷰·마케팅·홍보까지").
2. 우측 상단에 **✏️ 작은 연필 배지**가 나타납니다.
3. 클릭하면 모달이 열리고 → 새 텍스트 입력 → "저장".
4. 저장하는 즉시 라이브에 반영됩니다.

### 2.3 이미지 교체

1. 교체할 이미지에 마우스를 올림 → ✏️ 클릭.
2. 모달에서 "파일 선택" → 이미지 업로드.
3. 자동으로 WebP로 변환되고 저장됩니다.

### 2.4 링크 변경

CTA 버튼이 아닌 텍스트 링크(예: 푸터의 "이용약관")는 ✏️로 라벨과 URL을 바꿀 수 있습니다.

### 2.5 변경 히스토리 (롤백)

- 모든 편집은 **직전 5개 버전**까지 자동 저장됩니다.
- 잘못 편집했다면 super_admin에게 요청 → DB에서 이전 버전으로 복원 가능.

> ⚠️ **주의** : 인라인 편집은 텍스트와 이미지 교체용입니다. 섹션 추가/삭제 같은 구조 변경은 개발팀 작업이 필요합니다.

> 💡 **CTA 버튼**(예: "지금 신청하기")은 인라인 편집이 아니라 **CTA 관리** 메뉴에서 통합 관리합니다 (9번 섹션).

## 4. CRM — 상담 신청 관리

신청 폼에서 들어온 모든 리드를 처리하는 핵심 메뉴입니다.

### 4.1 목록 화면

- 메뉴 → "상담 신청" 클릭
- 표 형태로 최신순 노출 (이름·연락처·상태·매체·등록일)
- 상단에 **상태별 필터** + **매체 필터** + **검색창**(이름·연락처·매장·메모)
- 매체 컬럼은 자동 분류 결과를 색상 배지로 표시 (보라=광고, 파랑=검색, 초록=자체 블로그, 주황=외부 블로그, 회색=직접)

### 4.1-1 🎯 유입 출처 펼침 (super_admin / admin / marketer 전용)

목록 우상단에 **"🎯 유입 출처 펼치기 ▶"** 토글이 있습니다 (counselor / viewer는 보이지 않음).

- 클릭하면 매체 컬럼 옆에 4개 컬럼이 추가로 등장:
  - **캠페인** : utm_campaign (예: brand-2026q2)
  - **키워드** : 광고 키워드 또는 검색어 (예: "카드단말기 0원")
  - **소재** : utm_content (예: hero-A)
  - **랜딩** : 자체 블로그 글 제목 또는 외부 도메인
- 한 번 펼치면 본인 브라우저에 기억되어 다음 진입 시에도 펼쳐져 있음 (다시 접기 가능)
- 권한 없는 사람은 토글 자체가 안 보임 → 데이터 분석가·마케터만 볼 수 있게 분리

### 4.2 상세 모달 — 행 클릭

표의 한 행을 클릭하면 모달이 열리고, 상단에 **🎯 유입 출처 카드**가 풀 너비로 표시됩니다 (모든 권한자에게 보임).

#### 유입 출처 카드 구성

| 항목 | 의미 |
|---|---|
| 매체 배지 | 자동 분류 결과 (네이버 광고 / 구글 검색 / 자체 블로그 / 직접 등) |
| 캠페인 | utm_campaign 그대로 |
| **키워드** | utm_term (광고) 또는 referer 추출 (네이버 검색) — 구글 organic은 `(not provided)` |
| **소재** | utm_content (광고 소재명/ID) |
| 랜딩 | 자체 블로그 글이면 글 제목, 외부면 도메인 |
| 원본 데이터 ▾ | 펼치면 utm 5종 + gclid + fbclid + referer 풀 URL 디버그용 |

키워드와 소재가 보이려면 광고대행사가 **utm_term / utm_content** 를 표준대로 박아야 합니다 → `docs/UTM_NAMING_GUIDE.md` 참조.

#### 그 외 모달 구성

| 왼쪽 (DB 정보) | 가운데 (고객 입력) | 오른쪽 (처리) |
|---|---|---|
| 등록일·IP·DB그룹 | 이름·연락처·메시지 | 상태 / 담당자 / 내부 메모 / ❤️ / 🚫 |

### 4.3 상태 변경

1. 우측 패널의 **상태 드롭다운** 클릭
2. 원하는 상태 선택 (예: "1차 통화 완료", "예약 확정", "거절")
3. (선택) 이력 메모 입력
4. "저장" → 상태 변경 + 이력 자동 기록

### 4.4 담당 상담사 배정

- 우측 **담당자 드롭다운**에서 상담사 선택
- counselor / admin / super_admin 권한자만 후보로 표시됩니다

### 4.5 내부 메모

- 고객에게 보이지 않는 내부 메모 작성용
- 통화 내용·상담 결과·다음 액션 기록
- 저장 시 작성자·시각이 같이 기록됩니다

### 4.6 ❤️ 즐겨찾기 / 🚫 블랙리스트

- **❤️ 즐겨찾기** : 유망 리드 표시. 즉시 토글됨.
- **🚫 블랙리스트** :
  - 어뷰징·장난 신청을 차단할 때 사용
  - 연락처와 IP를 옵션으로 차단 (다음에 같은 번호로 신청해도 등록되지 않음)
  - 차단 권한은 admin 이상

### 4.7 💰 매출(개통) 등록 — 핵심 기능

상담 모달 하단에 풀 너비 **💰 매출 카드**가 있습니다 (1리드 N매출 가능 — 단말기 + 인터넷 동시 가입 같은 케이스).

#### 자동 트리거 (개통완료 상태로 변경 시)

상담 상태를 `is_conversion = true` 인 상태(예: "개통완료")로 변경하면, 저장 직후 **매출 등록 모달이 자동으로 뜹니다**.

- "건너뛰기" 가능 (상태만 변경, 매출은 나중에 입력)
- 등록 시 상품 선택 → 기본 매출액·약정·월 매출액이 자동 채움 (수정 가능)

#### 매출 입력 항목

| 항목 | 의미 |
|---|---|
| **상품** | 드롭다운에서 등록된 상품 선택 (선택 안 함도 가능 — 직접 입력) |
| **매출액** | 총 매출 (KRW) |
| **사은품액** | 통신사 지원금·기타 지원 — **매출에서 차감**되어 순매출 자동 계산 |
| **약정 기간** | 12/24/36/48개월 |
| **월 매출** | 구독형 상품 월 단가 (선택) |
| **개통일** | 실제 매출 발생일 |
| **메모** | 설치 일정·고객 특이사항 |

#### 권한

- **counselor 이상** 등록·편집 가능 (상담사가 직접 입력)
- **admin / super_admin** 만 삭제 가능

#### 누적 표시

매출 카드 하단에 누적 합계가 자동 계산:
- 누적 매출 / 사은품 / 순매출
- 첫 신청 → 첫 매출 소요일 (코호트 분석 핵심)

### 4.8 ◀ ▶ 페이지네이션 단축키

- 상세 모달이 열린 상태에서 **← → 키보드** 로 이전/다음 신청 이동
- **Esc** 키로 모달 닫기

### 4.8 이력 조회

- 모달 하단에 상태 변경 이력 + 메시지 발송 이력이 자동 표시
- 누가·언제·어떤 상태로 바꿨는지 모두 기록됩니다

> 💡 **상담사용 추천 워크플로우** :
> 1. 출근 → "상담 신청" 메뉴
> 2. 상태 "신규" 필터로 필터링
> 3. 위에서부터 행 클릭 → 통화 → 메모 작성 → 상태 변경
> 4. → 키로 다음 건으로 이동 (반복)

## 5. 콘텐츠 — 블로그 글쓰기

자영업 사장님 대상 SEO 블로그 글을 작성·관리하는 메뉴입니다.

### 5.1 목록 화면

- 메뉴 → "콘텐츠"
- 표 (제목·카테고리·SEO 점수·상태·조회수·수정일)
- 📌 표시 = 목록 상단 고정 글
- 우상단 **+ 새 글 작성** 버튼

### 5.2 새 글 작성 — 화면 구성

```
┌──────────────────────────────┬──────────────────┐
│ 제목 입력                     │ 🎯 SEO 어시스턴트│
│ slug (자동 생성, 수정 가능)    │ ───────────────  │
│ ┌─ TipTap 툴바 ────────────┐ │ 검색 미리보기     │
│ │ H2 H3 H4 / B I S         │ │ 포커스 키워드     │
│ │ • 목록 / 1. 목록 / 인용  │ │ 종합 점수 + 등급 │
│ │ 코드 / 링크 / 🖼 이미지   │ │ Tier 1 SEO       │
│ └───────────────────────────┘ │ Tier 2 다이아     │
│ 본문 에디터                   │ Tier 3 EEAT       │
│                              │                  │
│ 요약 / 카테고리 / 태그        │ OG 미리보기      │
│ 작성자 / 📌 고정             │ 본문 통계         │
│ 메타 제목 / 설명 / 대표 이미지│                  │
│                              │                  │
│ [임시저장] [발행]             │                  │
└──────────────────────────────┴──────────────────┘
```

### 5.3 TipTap 에디터 사용법

| 버튼 | 단축 | 기능 |
|---|---|---|
| H2 / H3 / H4 | — | 소제목 (글 구조화 — SEO 핵심) |
| **B** | Cmd/Ctrl+B | 굵게 |
| *I* | Cmd/Ctrl+I | 기울임 |
| ~~S~~ | — | 취소선 |
| • 목록 / 1. 목록 | — | 비순서/순서 리스트 |

### 5.4 SEO 어시스턴트 — 3-Tier 자체 점수

오른쪽 패널에서 실시간으로 점수가 계산됩니다.

#### 종합 등급

| 등급 | 점수 |
|---|---|
| **S** | 85점 이상 |
| **A** | 70~84점 |
| **B** | 50~69점 |
| **C** | 50점 미만 |

#### Tier 1 — 기본 SEO (40% 가중)

RankMath 류 7항목. 포커스 키워드를 입력해야 계산이 시작됩니다.

- 제목(H1)에 키워드 포함 (20점)
- 첫 150자에 키워드 (15점)
- 키워드 밀도 0.5~3% (15점)
- 메타 설명에 키워드 (10점)
- 소제목 H2/H3 사용 (15점)
- 이미지 alt에 키워드 (10점)
- 내부 링크 포함 (15점)

#### Tier 2 — 네이버 다이아 휴리스틱 (30% 가중)

- 본문 1500자 이상 (20점)
- 이미지 2개 이상 (15점)
- H2 소제목 3개 이상 (15점)
- 표 또는 리스트 사용 (10점)
- 외부 권위 사이트 인용 — naver/go.kr 등 (10점)
- 키워드 자연 분포 1~2% (15점)
- 한 문장 평균 60자 이하 (모바일 가독성) (15점)

#### Tier 3 — 구글 EEAT (30% 가중)

- **E**xperience : "직접", "실제로", "사례" 등 경험어 2개 이상 (25점)
- **E**xpertise : 수치·단위 (%, 원, 건) 3개 이상 (25점)
- **A**uthoritativeness : 외부 출처 링크 1개 이상 (25점)
- **T**rustworthiness : 작성자명·업데이트일·"출처/참고" 키워드 중 2개 이상 (25점)

> 💡 **만점 노리는 글쓰기 팁** :
> 1. 본문 1500자 이상 + H2 3개 이상 + 이미지 2개 이상 → 다이아 만점 기반
> 2. 포커스 키워드를 제목·첫 문단·메타 설명에 자연스럽게 모두 포함
> 3. "직접 운영해본 결과 매출이 1.7배 늘었습니다" 같은 경험·수치 표현
> 4. 네이버 페이 공식 사이트, 정부 보도자료 등 권위 사이트 링크 1~2개
> 5. 작성자명 채우고 본문에 "출처:" 인용 1줄

### 5.5 메타 태그 입력

- **SEO 제목** : 비우면 글 제목 그대로 사용. 30~60자 권장 (구글 노출 기준)
- **메타 설명** : 검색 결과에 노출. 120~160자 권장
- **대표 이미지** : 카카오톡·페이스북 공유 시 노출. 직접 URL 입력 또는 "업로드" 버튼

### 5.6 카테고리

| 코드 | 라벨 | 용도 |
|---|---|---|
| guide | 가이드 | 사장님용 노하우 글 |
| case_study | 사례 | 도입 매장 케이스 스터디 |
| blog | 블로그 | 일반 블로그 |
| news | 뉴스 | 회사 소식·언론 |
| faq | FAQ | 자주 묻는 질문 (검색용) |

### 5.7 태그

콤마로 구분 입력 (예: `POS, 카드단말기, 자영업`). 라이브 글 하단에 #태그로 표시됩니다.

### 5.8 임시저장 vs 발행

- **임시저장** : DB에 저장되지만 라이브에는 노출 X. 작성 중간 백업용
- **발행** : 즉시 라이브 반영. `/blog` 목록과 sitemap에 자동 포함
  - 처음 발행 시 `published_at`이 기록됨 (이후 수정해도 보존)

### 5.9 글 수정·삭제

- 목록에서 글 제목 클릭 → 에디터 열림 → 수정 후 "발행"
- 발행된 글 수정 시 → 라이브 캐시 자동 무효화 → 즉시 반영
- "삭제" 버튼 (admin / super_admin만 보임) → 라이브에서도 즉시 사라짐 → 복구 불가

### 5.10 📌 고정 (목록 상단)

- 에디터 우측 "📌 목록 상단에 고정" 체크
- 가장 중요한 글(예: "단말기 0원 진실") 1~2개에 사용
- 고정 글이 항상 `/blog` 최상단에 노출

## 7. 관리자 추가·관리

> 👑 **super_admin 전용 메뉴**입니다.

### 7.1 새 관리자 초대 (이메일 링크)

1. 메뉴 → "사용자 관리"
2. 우상단 **+ 사용자 초대** 클릭
3. 이메일 입력 → 권한 선택 (counselor / marketer / admin / super_admin) → 표시 이름 입력
4. "초대 메일 발송" → 상대방에게 가입 링크가 자동 발송됨
5. 상대방이 링크로 가입 → 자동으로 권한이 부여되고 어드민 진입 가능

### 7.2 권한 변경

- 사용자 행 클릭 → 권한 드롭다운에서 변경 → "저장"
- 즉시 반영. 상대방이 새로고침하면 새 권한으로 동작

### 7.3 비활성화 vs 삭제

| 동작 | 효과 | 사용 케이스 |
|---|---|---|
| **비활성화** | `is_active = false`, 로그인 차단, 데이터 보존 | 휴직·일시 정지 |
| **삭제** | admin_users 행 삭제 (auth는 유지될 수 있음) | 영구 퇴사 |

### 7.4 퇴사자 처리 추천 흐름

1. 해당 상담사가 담당하던 상담을 다른 상담사에게 **재배정** (사용자 관리 → "이관" 버튼)
2. **비활성화** 처리 (계정 삭제는 즉시 X — 30일 정도 보존 권장)
3. 1개월 후 문제없으면 **삭제**

### 7.5 super_admin 보호 규칙

다음은 시스템이 자동으로 막습니다 (실수 방지):

- **본인을 강등할 수 없음** (다른 super_admin이 해야 함)
- **본인을 비활성화할 수 없음**
- **마지막 활성 super_admin은 비활성화·삭제·강등 불가** (회사가 잠기는 것을 방지)

> 💡 super_admin은 최소 2명을 유지하는 것을 강력 권장합니다 (1명이 퇴사·휴가 시 잠김 방지).

### 9.1 CTA 위치 (placement)

| 위치 코드 | 어디에 보이나 |
|---|---|
| nav | 상단 네비 우측 |

1. 메인 카피 위 hero CTA에 캠페인별 utm 다르게 설정
2. 광고 매체별로 CTA 라벨 A/B 테스트 (예: "지금 신청" vs "무료 상담")
3. 1주일 단위로 대시보드에서 전환율 비교 → 승자 채택

## 10. 대시보드 보는 법

메뉴 → "대시보드" (또는 어드민 진입 시 기본 화면)

### 10.1 상단 KPI 카드

- 오늘/이번 주/이번 달 신청 수
- 유망 리드 수 (is_promising)
- 전환 수 (is_conversion)
- 반려 수 (is_unapproved)

### 10.2 상태별 분포

13개 상태마다 신청 건수 + 색상 배지로 한눈에 확인.

### 10.3 매체별 전환율 (자동 5분류)

`v_consultations_by_inferred_channel` 뷰 기준. 단순 utm_source가 아닌 **자동 분류 결과** 기준입니다:

| 분류 | 의미 | 색상 |
|---|---|---|
| `naver-ads` / `google-ads` / `meta-ads` 등 | 광고 캠페인 (utm_medium에 -ads/cpc 들어간 것) | 보라 |
| `naver-search` / `google-search` 등 | 검색 자연 유입 (referer가 검색엔진) | 파랑 |
| `internal-blog` | **자체 블로그 → 신청** (콘텐츠 ROI 핵심) | 초록 |
| `referral-blog` | 외부 블로그·카페 (네이버 블로그·티스토리·브런치) | 주황 |
| `social-organic` | SNS 자연 공유 (페북·인스타·유튜브) | 분홍 |
| `kakao` | 카카오톡 메시지 | 노랑 |
| `referral-other` | 그 외 외부 사이트 | 베이지 |
| `direct` | utm·referer 둘 다 없음 (북마크·직접입력·인앱브라우저) | 회색 |

각 분류별: 신청 수 / 오늘·1주·1개월 추이 / 전환 수.

> 💡 **해석 포인트** :
> - `direct` 비율이 30% 넘으면 → utm 누락된 트래픽 많거나 카톡 인앱 브라우저 referer 유실 의심
> - `internal-blog` 비율이 15% 넘으면 → 콘텐츠 마케팅 ROI 양호. 더 발행 가치
> - 광고 매체별 전환율이 1% 미만 → 소재·랜딩 점검 필요
> - 구글 organic 키워드는 `(not provided)` — Search Console 별도 조회

### 10.3-1 키워드 / 소재 Top 분석

DB 뷰 `v_consultations_by_keyword`, `v_consultations_by_creative` 로:

- 광고 키워드별 신청 수·전환율 → 입찰가 조정 근거
- 광고 소재(A/B 테스트)별 성과 비교 → 승자 소재 채택

### 10.3-2 자체 블로그 글별 ROI (콘텐츠 마케팅)

뷰 `v_consultations_by_blog_post` :

- 어떤 블로그 글에서 신청이 많이 들어왔는지
- 글 1개당 신청 수·전환 수
- 5위 안에 못 든 글은 콘텐츠 개선 또는 발행 중단 검토

### 10.4 CTA별 성과 표

CTA 위치·라벨별로:
- 클릭 → 신청 → 전환의 단계별 수치
- 1위 CTA를 기본 채택, 부진 CTA는 비활성화 또는 교체

### 10.5 최근 5건

가장 최근 들어온 신청 5건 → 클릭하면 즉시 상세 모달 진입

- [ ] 상담사 최소 1명 활성 + 본인 권한 확인
- [ ] 대시보드 KPI 카드 정상 작동 확인
- [ ] 모바일에서 신청 폼 직접 한 번 제출해보기

### 11.2 신청 알림 흐름

신청 폼 제출 → DB INSERT → 슬랙 알림 자동 발송 → 상담사가 어드민에서 처리

### 11.5 유입 출처별 운영 가이드

#### `direct` (직접 진입) 비율이 높을 때

원인 후보:
- 광고 URL에 utm 파라미터 누락 → 광고대행사한테 `UTM_NAMING_GUIDE.md` 전달
- 카카오톡 메신저 인앱 브라우저는 referer 유실 → 별도 추적 어려움 (어쩔 수 없음)
- 명함·전단지 인쇄물 → 별도 utm 부여 필요 (예: `?utm_source=offline&utm_medium=flyer&utm_campaign=2026q2-flyer`)

#### `internal-blog` 비율 늘리는 콘텐츠 전략

자체 블로그 → 신청 전환은 가장 저렴한 리드 (광고비 0원). 이 비율을 키우는 게 장기 ROI 핵심:

1. SEO S 등급 글을 매주 1편 발행 (3-Tier 점수 시스템 활용)
2. 글 본문·하단에 신청 폼 CTA 항상 배치
3. 자체 블로그에서 신청한 사람 = 이미 우리 콘텐츠를 읽고 신뢰한 상태 → 전환율 일반적으로 광고 대비 2~3배 높음

#### 광고 매체별 CPL (Cost Per Lead) 추적

어드민에는 광고비 데이터가 없으니, 매체별 신청 수만 보여줌. 광고비는 매체 대시보드(네이버 검색광고 / Meta Ads Manager / Google Ads)에서 확인:

```
CPL = 매체별 광고비 ÷ 어드민 매체별 신청 수
```

기준선: 자영업자 단말기 신청 CPL 5,000~15,000원 권장 (업계 평균).

#### UTM 표준 가이드 (광고대행사 필독)

`docs/UTM_NAMING_GUIDE.md` 참고. 매체별 셋업 표준 + 자주 틀리는 5가지 + 테스트 방법까지 정리되어 있음.

### 11.6 SEO 점수 등급 운영 기준

- **C 등급** : 발행 비추천. 다이아·EEAT 보강 후 재시도
- **B 등급** : 임시저장만. 한 번 더 손보고 발행
- **A 등급** : 발행 OK. 추가 보강 권장
- **S 등급** : 발행 적극 권장. 광고 랜딩으로 활용 가능

## 12. 트러블슈팅

### 12.1 로그인이 안 돼요

- 비밀번호 재설정 : 로그인 페이지 → "비밀번호 찾기"
- super_admin이 계정을 비활성화했을 수 있음 → super_admin에게 문의
- "no_access" 에러 : admin_users에 등록되지 않음 → super_admin에게 권한 부여 요청

### 12.2 인라인 편집 ✏️ 버튼이 안 보여요

- 같은 브라우저에 어드민 로그인이 되어 있는지 확인
- 권한이 viewer / counselor라면 편집 권한 없음 → marketer 이상 필요
- 캐시 문제 : 강력 새로고침 (Cmd+Shift+R / Ctrl+Shift+R)

### 12.3 이미지 업로드가 실패해요

- 파일 크기 5MB 초과 → 압축 후 재시도
- 지원 형식 : jpg / png / gif / webp만 가능
- 권한 부족 : viewer / counselor는 업로드 불가

### 12.4 발행했는데 라이브에 안 보여요

- 1~2분 대기 (CDN 캐시 갱신 시간)
- 강력 새로고침
- super_admin에게 문의 (Vercel 빌드 실패 가능성)

### 12.5 상담 모달이 안 열려요

- 다른 사용자가 같은 행을 편집 중일 수 있음 → 잠시 후 재시도
- 새로고침 후 재시도
- 계속되면 super_admin에게 문의

### 12.6 SEO 점수가 0점으로 나와요

- 우측 패널의 **포커스 키워드**를 입력해야 점수 계산이 시작됩니다
- 포커스 키워드는 1~3단어 추천 (예: "카드 단말기 0원")

### 12.7 슬랙 알림이 안 와요

- 슬랙 워크스페이스에서 #leads 채널이 살아있는지 확인
- super_admin이 webhook 환경변수 점검 필요

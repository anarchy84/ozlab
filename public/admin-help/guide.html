2 상담 신청 처리

핵심 화면. 신청 들어온 모든 리드를 처리하는 곳.

기본 흐름

  1. 사이드바 "상담 신청" 클릭
  2. 리스트에서 행 클릭 → 상세 모달
  3. 모달에서 상태 변경 / 메모 작성 / 담당자 배정
  4. ← → 키로 다음 건 이동
  5. Esc 닫기
/admin/consultations — 행 어디든 클릭하면 모달

모달 안에서 할 수 있는 것

상태 변경드롭다운에서 "개통완료" 등 선택 → 자동 매출 모달 뜸
담당 상담사 배정counselor 권한 사용자 중 선택
내부 메모고객은 못 봄. 통화 결과·다음 액션 기록
❤️ 즐겨찾기유망 리드 마킹 (즉시 토글)
🚫 블랙리스트장난·어뷰징 신청 차단 (연락처/IP)
이력 조회모달 하단 — 누가 언제 무엇을 변경했는지
💡 상담사 워크플로우 : 출근 → "신규" 필터 → 위에서부터 행 클릭 → 통화 → 메모 + 상태 변경 → → 키로 다음 건. 반복.

3 유입 출처 보기

이 리드가 어디서 들어왔는지 자동 분류돼서 한 줄로 보입니다.

매체 컬러 코드

🟣 보라광고 (네이버·구글·메타 광고)
🔵 파랑검색 자연 유입 (네이버·구글 검색)
🟢 초록자체 블로그 → 신청 (콘텐츠 ROI 핵심)
🟠 주황외부 블로그·카페 (네이버 블로그·티스토리)
⚫ 회색직접 진입 (북마크·카톡 인앱)

펼침 패널 (마케팅팀·TM실장·슈퍼어드민만 보임)

상담 목록 우상단 "🎯 유입 출처 펼치기 ▶" 토글 → 캠페인·키워드·소재·랜딩 4컬럼 추가.

펼침 토글 ON → 캠페인/키워드/소재/랜딩 4컬럼 추가 (초록 = 자체 블로그 ROI 핵심)
⚠️ 광고 utm 표준 안 박으면 분류가 깨집니다. 광고대행사한테 UTM_NAMING_GUIDE.md 전달하세요.

4 매출 등록 (가장 중요)

리드별로 매출을 기록해야 광고 ROAS·LTV·코호트 분석이 작동합니다.

자동 트리거

상담 상태를 "개통완료" 같은 전환 상태로 변경하면 → 매출 입력 모달이 자동으로 뜸.

입력 항목

상품드롭다운에서 선택 (Npay커넥트단말기 등)
매출액총 매출 (원)
사은품액통신사 지원금·기타 — 매출에서 차감되어 순매출 자동
약정12/24/36/48개월
월 매출구독형 상품 (선택)
개통일실제 매출 발생일
💡 한 리드에 여러 매출 기록 가능 (단말기 + 인터넷 동시 가입 등). 모달 우측 매출 카드에 누적 합계 자동.
⚠️ 매출 입력은 상담사 책임. 안 넣으면 ROAS·LTV가 0으로 나와 광고 효율 측정 불가.
상태를 "개통완료"로 변경하면 자동으로 뜸 — 사은품(노란색)은 매출에서 차감되어 순매출 자동 계산

5 광고 시트 동기화

루커 스튜디오용 Google Sheets를 어드민으로 가져옴 → 매출과 자동 매칭.

한 번 설정

  1. Google Sheets 열고 파일 → 공유 → 웹에 게시 → CSV 형식 → 게시
  2. 나오는 URL 복사
  3. 어드민 → "광고 sync" → URL 붙여넣기 → 저장
  4. "🔄 지금 동기화" 클릭

시트 컬럼 (1행 헤더)

한글/영문 모두 OK. 둘 중 하나만 있으면 자동 매핑.

한글영문예시
날짜 / 일자date2026-04-29
매체 / 채널channelnaver / google / meta / daangn
서비스 / 상품군service토스단말기 / 인터넷가입 (선택)
노출수impressions5217
클릭수clicks151
전환수conversions26
광고비 / 비용spend1,584,970
💡 같은 (날짜·매체·서비스) 행은 덮어쓰기(UPSERT). 시트 갱신 후 sync 다시 누르면 최신값으로 갱신됨.
⚠️ 자동 cron 동기화는 다음 버전 추가 예정. 현재는 수동 클릭만.

6 매출·광고 대시보드

매출 분석 (경영자 뷰)

사이드바 "매출 분석" → 총매출 / 총광고비 / ROAS / 매체별 코호트 (30/60/90일 누적) / 매체별 LTV.

광고 성과 (마케터 뷰)

사이드바 "광고 성과" → 노출/클릭/CTR/전환/CVR/CPA/광고비 + 매체별 합계.

💡 둘 다 시트 sync + 매출 등록이 돼야 데이터가 나옴. 둘 중 하나만 비어 있으면 한쪽 KPI는 0.

7 블로그 글쓰기

SEO 최적화 글 작성. 자체 블로그 → 신청 전환은 광고비 0원의 황금 채널.

흐름

  1. 사이드바 "콘텐츠" → "+ 새 글 작성"
  2. 제목 입력 (slug 자동 생성)
  3. TipTap 에디터로 본문 (🖼 이미지 즉석 업로드)
  4. 오른쪽 패널 포커스 키워드 입력 → 3-Tier SEO 점수 자동
  5. 임시저장 또는 발행

3-Tier SEO 점수

Tier 1 (40%)기본 SEO — RankMath류 7항목 (제목·키워드 밀도·메타·이미지 alt 등)
Tier 2 (30%)네이버 다이아 — 본문 1500자+ / H2 3개+ / 표·리스트 / 권위 인용 / 모바일 가독성
Tier 3 (30%)구글 EEAT — 직접 경험어 / 수치·단위 / 외부 출처 / 작성자·출처
💡 S 등급(85점+) 노리는 팁 : 1500자 + 이미지 2개 + H2 3개 + 외부 권위 사이트 1개 인용 + "직접 운영" 같은 경험어 + 수치 3개 이상.

8 홈 인라인 편집 · CTA 관리

홈 인라인 편집

어드민 로그인 상태에서 홈(ozlabpay.kr) 방문 → 텍스트·이미지에 마우스 올리면 ✏️ 작은 연필 배지 노출 → 클릭 → 모달로 즉석 편집.

CTA 관리 (슈퍼어드민)

홈의 모든 "지금 신청하기" 버튼을 한 곳에서 관리. 위치·라벨·링크·utm 파라미터 자동 부여.

  • 매체별 utm 다르게 설정 → 어트리뷰션 자동 추적
  • A/B 테스트 시 라벨만 바꿔서 클릭률 비교
  • 대시보드에서 CTA별 전환율 표 확인

10 사용자 추가 (슈퍼어드민)

  1. 사이드바 "사용자 관리" → "+ 사용자 초대"
  2. 이메일 + 권한 (마케팅팀/TM실장/상담사/전산관리자) + 표시 이름
  3. 초대 메일 자동 발송 → 상대방이 가입 → 자동 권한 부여

퇴사 처리

  1. 해당 상담사 담당 리드를 다른 상담사에게 재배정
  2. 비활성화 (계정 삭제 X — 데이터 보존)
  3. 1개월 후 문제없으면 삭제
⚠️ 슈퍼어드민 보호 : 본인 강등·비활성 불가, 마지막 슈퍼어드민 삭제 불가. 잠김 방지를 위해 슈퍼어드민 최소 2명 유지 권장.

11 상품 카탈로그 (마케팅팀·슈퍼어드민)

매출 입력 모달의 드롭다운에 노출될 상품 관리.

좌측 카테고리 (추가/수정/삭제 가능)

Pos (카드단말기) / 인터넷가입 / 테이블오더 / CCTV / 키오스크

우측 상품

  • 코드 (영문, 변경 불가) · 상품명 · 카테고리
  • 기본 매출액 · 약정 · 월 구독액 → 매출 입력 시 자동 채움
  • 활성/비활성 토글

12 DB 자동분배 (마케팅팀·TM실장)

새 리드 들어올 때 라운드로빈으로 상담사에게 공평 자동 배정.

설정

  1. 사이드바 "DB 분배"
  2. "자동 배정 활성" 토글 ON
  3. 분배 방식: 라운드로빈 (공평 순환)
  4. 배정 대상 role 선택 (보통 "상담사")

재분배

  • 미배정 리드만 재분배 — 안전. 배정 안 된 것만 처리
  • 전체 재분배 (덮어씀) — 주의. 기존 배정 다 무시하고 다시
⚠️ 가중치 / 시간대별 / 일일 한도 옵션은 다음 버전 추가 예정.

Q. 인라인 편집 ✏️ 버튼이 안 보여요

A. 같은 브라우저에 어드민 로그인이 돼있는지 확인. counselor 권한은 편집 불가 — 마케팅팀 이상 필요. 강력 새로고침 (Cmd+Shift+R).

Q. 매출 등록했는데 대시보드에 안 보여요

A. 1~2분 기다린 뒤 새로고침. 매출 분석 대시보드는 캐시 없이 매번 fresh. 그래도 안 보이면 상담사가 다른 리드의 매출로 잘못 등록한 건 아닌지 확인.

Q. 광고 시트 sync 가 0행으로 나와요

A. 시트 1행 헤더가 date, channel, ... 또는 한글 동의어인지 확인. 시트가 "웹에 게시" 상태인지 확인 (그냥 공유는 안 됨). URL 끝이 output=csv 인지.

Q. 자동 매출 모달이 안 떠요

A. 변경한 상태가 is_conversion=true 인 상태인지 확인 (상태 관리 메뉴). 기본 시드는 "개통완료" 같은 전환 상태에만 플래그 ON.

Q. 권한 변경했는데 반영이 안 돼요

A. 해당 사용자가 어드민에서 새로고침 1회. 그래도 안 되면 로그아웃 → 재로그인.

Q. utm 표준 어디에 있어요?

A. docs/UTM_NAMING_GUIDE.md — 광고대행사한테 그대로 전달 가능한 양식.
